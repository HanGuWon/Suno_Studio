# ARA plan (future phase)

## Why ARA later

ARA can improve context reading (region-aware audio access, tempo/key context, clip-follow behavior) in supported hosts.

## What may move to ARA mode

- Region-aware source capture for audio prompts.
- Better synchronized context exchange with host timeline.

## What remains host-specific even with ARA

- True media insertion/placement automation.
- Track/selection manipulation APIs.

These remain per-host adapter responsibilities.
